## 1. Purpose

This document defines the **voice, tone, and microcopy rules** for NEXUS. Every string in the UI must follow these rules. It also establishes a small microcopy pattern library for common UX moments.

## 2. Voice

NEXUS has a single voice:

| We are | We are not |
|--------|-----------|
| Calm | Hype-y |
| Direct | Condescending |
| Specific | Vague |
| Confident | Cocky |
| Honest about uncertainty | Overpromising |
| Friendly without being cute | Cartoonish |

## 3. Tone

Tone varies by context:

| Context | Tone |
|---------|------|
| Empty states | Inviting, gentle |
| Errors | Calm, helpful |
| Successes | Brief, satisfied |
| Onboarding | Encouraging, simple |
| Permissions | Honest, transparent |
| Loading | Brief, informative |

Tone is implemented by word choice and length, not punctuation.

## 4. Voice rules

1. **Active voice.** "Save changes" not "Changes can be saved."
2. **Present tense.** "Saving…" not "Will save…"
3. **Second person.** "Your workspace is ready" not "The workspace is ready."
4. **Avoid jargon.** "Sign in" not "Authenticate via OAuth provider."
5. **One sentence per idea.** Break long sentences.
6. **No exclamation marks.** (Except celebrations, sparingly.)
7. **No emoji in product UI.** (Optional in onboarding.)
8. **Avoid filler.** "Just," "simply," "easily" — drop.

## 5. Microcopy patterns

### 5.1 Buttons

| Context | Pattern | Example |
|---------|---------|---------|
| Primary action | Verb (present) | "Save changes" |
| Destructive | Verb + object | "Delete workspace" |
| Cancel | Cancel | "Cancel" |
| Confirm in dialog | Verb + object | "Yes, delete workspace" |
| Disabled | Verb (passive) | "Saving…" |

Action buttons always have verb labels. Generic "OK" is forbidden.

### 5.2 Empty states

Format: **illustration + headline + description + primary CTA + secondary link**

```
[Illustration]

No workspaces yet

Workspaces organize your work by goal, not by tab.
Create your first workspace to get started.

[Create workspace]   Browse templates
```

Headlines are short (≤6 words). Descriptions are 1–2 sentences.

### 5.3 Errors

Format: **icon + headline + description + recovery + support link**

```
[⚠ icon]

Couldn't save your changes

We had trouble reaching the server. Your changes are
preserved locally and will retry automatically.

[Retry now]   Get help
```

Errors are calm. They tell the user what happened, what they can do, and where to get more help.

### 5.4 Loading

| Duration | Pattern |
|----------|---------|
| <500ms | No indicator |
| 500ms–2s | Skeleton |
| 2–10s | Skeleton + label ("Generating…") |
| >10s | Skeleton + label + cancel option |

### 5.5 Confirmations

| Action | Pattern |
|--------|---------|
| Low-stakes | Toast, auto-dismiss |
| Medium-stakes | Inline message, persistent |
| High-stakes | Modal, requires confirm |

### 5.6 Permission prompts

Format: **who is asking + what it wants + why + how to revoke**

```
LinkedIn Outreach wants to:

  ✓ Read your inbox
  ✓ Send messages on your behalf

This is needed to find and contact your prospects.
You can revoke these permissions anytime in Settings.

[Approve]   [Deny]
```

Permissions are never vague. Lists are exhaustive.

### 5.7 AI responses

| Pattern | When |
|---------|------|
| Streaming | Default; tokens appear as generated |
| Inline artifact | When output is structured (table, chart) |
| Confidence hint | When uncertain; e.g., "I'm not certain, but…" |

### 5.8 Notifications

| Type | Pattern |
|------|---------|
| Informational | Brief description + action link |
| Action required | Verb-led headline + action |
| Success | Past-tense verb + object ("Workflow scheduled.") |

## 6. Honesty rules

1. **Agents are labeled agents.** "Planner agent says…" not "NEXUS thinks…"
2. **Confidence is shown.** "Probably…" / "I think…" / "I'm not sure…"
3. **Failures are owned.** "Couldn't find what you asked for" not "No results."
4. **Limitations are admitted.** "This feature requires Pro" not "Upgrade now."
5. **No fake testimonials.** Marketing copy cites real users.

## 7. Microcopy in agent output

When agents produce text:

- Generated content is marked: "Generated by [agent name]"
- Inconclusive responses say so.
- Citations are inline ("Source: Example.com").
- Confidence is shown when below threshold.

## 8. Truncation and ellipsis

| Pattern | Usage |
|---------|-------|
| `…` | Mid-sentence truncation |
| `Show more` | Expandable content |
| `Read more` | Long-form articles |
| `[+N more]` | List overflow |

Never use `...` (three dots); use the typographic ellipsis `…`.

## 9. Acceptance criteria

- [ ] All user-facing strings pass a voice review.
- [ ] No jargon in end-user surfaces.
- [ ] Permission prompts always list exhaustive actions.
- [ ] Errors always include recovery path.
- [ ] Loading indicators match duration.
- [ ] Empty states follow the pattern.
- [ ] AI responses are labeled.
